or wherever repo is cloned\n\n# Pull latest code (main branch with Phase 4 cleanup merged)\ngit fetch origin\ngit checkout main\ngit pull origin main\n\n# Verify Phase 4 cleanup is present\nls -la OLLAMA-MIGRATION.md OLLAMA-INTEGRATION.md PHASE-4-CLEANUP-IMPLEMENTATION.md\n\n# Verify duplicate code is removed\n[ ! -d extensions/ollama-chat ] && echo \"✅ ollama-chat removed\" || echo \"❌ ollama-chat still present\"\n[ ! -d backend/src/services/ai ] && echo \"✅ AI services removed\" || echo \"❌ AI services still present\"\n[ ! -f scripts/ollama-init.sh ] && echo \"✅ ollama-init.sh removed\" || echo \"❌ ollama-init.sh still present\"\n```\n\n### Step 3: Verify Ollama Repository Availability\n\n```bash\n# Option A: Ollama as git submodule (if configured)\ngit submodule update --init --recursive\n\n# Option B: Ollama cloned separately for development\ncd /opt\nif [ ! -d ollama ]; then\n  git clone https://github.com/kushin77/ollama.git\nelse\n  cd ollama\n  git fetch origin\n  git checkout main\n  git pull origin main\nfi\n```\n\n### Step 4: Stop Running Services\n\n```bash\n# Stop code-server and related services\ncd /opt/code-server-enterprise\ndocker-compose down\n\n# Verify services are stopped\ndocker-compose ps\necho $?\n```\n\n**Expected**: All services show as \"stopped\" or missing.\n\n### Step 5: Update Environment Variables\n\n```bash\n# Update .env with necessary configuration\ncat >> .env << 'EOF'\n\n# Ollama Configuration (Phase 5 Deployment)\nOLLAMA_ENDPOINT=http://ollama:11434\nOLLAMA_DEFAULT_MODEL=llama2:7b-chat\nOLLAMA_NUM_THREAD=8\nOLLAMA_NUM_GPU=0  # Set to 1+ if GPU available\n\n# AI Services Configuration\nAI_INDEXING_ENABLED=true\nAI_INDEX_CHUNK_SIZE_TOKENS=800\nAI_INDEX_OVERLAP_TOKENS=120\nAI_INDEX_QUEUE_SIZE=1000\n\n# Leave these unset unless needed:\n# AI_EGRESS_ENABLED=false\n# HF_API_TOKEN=\n\nEOF\n\n# Verify configuration\ngrep -E \"OLLAMA|AI_\" .env\n```\n\n### Step 6: Rebuild Docker Images\n\n```bash\n# Rebuild code-server image with Phase 4 changes\ndocker-compose build --no-cache code-server\n\n# Pull latest ollama image\ndocker-compose pull ollama\n\n# Verify images\ndocker images | grep -E \"code-server|ollama\"\n```\n\n### Step 7: Start Services\n\n```bash\n# Start services in order: ollama first, then code-server\ndocker-compose up -d ollama\nsleep 10  # Give ollama time to start\n\n# Verify ollama is running\ndocker-compose ps ollama\n\n# Check ollama health\ncurl -s http://localhost:11434/api/tags | jq '.models | length' || echo \"Waiting for ollama...\"\nsleep 5\n\n# Start code-server\ndocker-compose up -d code-server\n\n# Start other services\ndocker-compose up -d\n\n# Verify all services\ndocker-compose ps\n```\n\n### Step 8: Verify Service Health\n\n```bash\n# Check code-server logs\ndocker-compose logs --tail 50 code-server\n\n# Check ollama logs\ndocker-compose logs --tail 50 ollama\n\n# Verify code-server is accessible\necho \"Code-Server Health Check:\"\ncurl -s http://localhost:8080/health 2>/dev/null && echo \"✅ UP\" || echo \"❌ DOWN\"\n\n# Verify ollama is accessible\necho \"Ollama Health Check:\"\ncurl -s http://localhost:11434/api/tags | jq '.models' 2>/dev/null && echo \"✅ UP\" || echo \"❌ DOWN\"\n```\n\n### Step 9: Run Integration Tests on Host\n\n```bash\n# Optional: Run integration tests on deployed services\ncd /opt/code-server-enterprise/backend\nOLLAMA_ENDPOINT=http://localhost:11434 npm run test:integration 2>&1 | tail -20\n```\n\n## Post-Deployment Validation\n\n### Verification Checklist\n\n```bash\n#!/bin/bash\n# Save as: /opt/code-server-enterprise/scripts/verify-phase-5-deployment.sh\n\necho \"Phase 5 Deployment Verification\"\necho \"=====================================\"\necho \"\"\n\n# 1. Check services are running\necho \"1. Service Health:\"\ndocker-compose ps | grep -E \"(code-server|ollama|caddy|postgres|redis)\"\necho \"\"\n\n# 2. Check code-server\necho \"2. Code-Server Check:\"\nCODE_SERVER_UP=$(curl -s -w \"%{http_code}\" -o /dev/null http://localhost:8080/health)\necho \"HTTP Status: $CODE_SERVER_UP\"\n[ \"$CODE_SERVER_UP\" = \"200\" ] && echo \"✅ Code-Server is UP\" || echo \"❌ Code-Server is DOWN\"\necho \"\"\n\n# 3. Check ollama\necho \"3. Ollama Check:\"\nOLLAMA_UP=$(curl -s http://localhost:11434/api/tags | jq -r '.models | length')\necho \"Models Available: $OLLAMA_UP\"\n[ -n \"$OLLAMA_UP\" ] && echo \"✅ Ollama is UP\" || echo \"❌ Ollama is DOWN\"\necho \"\"\n\n# 4. Docker Compose State:\"\ndocker-compose config --services\necho \"\"\n\necho \"Phase 5 Verification Complete!\"\n```\n\nRun the verification:\n```bash\nbash scripts/verify-phase-5-deployment.sh\n```\n\n### Browser Verification\n\n1. prometheus-rules.yml):\n\n```yaml\n- alert: OllamaDown\n  expr: up{job=\"ollama\"} == 0\n  for: 2m\n  annotations:\n    summary: \"Ollama service is down\"\n    action: \"Check docker-compose logs and restart ollama service\"\n\n- alert: OllamaHighLatency\n  expr: histogram_quantile(0.95, ollama_request_duration_ms) > 5000\n  for: 5m\n  annotations:\n    summary: \"Ollama high latency detected\"\n    action: \"Check resource usage and model parameters\"\n\n- alert: OllamaHighErrorRate\n  expr: rate(ollama_errors_total[5m]) > 0.05\n  for: 5m\n  annotations:\n    summary: \"Ollama error rate > 5%\"\n    action: \"Check ollama logs and verify models are loaded\"\n```\n\n## Rollback Procedure\n\nIf Phase 5 deployment has critical issues:\n\n### Quick Rollback (< 2 minutes)\n\n```bash\n# Stop all services\ncd /opt/code-server-enterprise\ndocker-compose down\n\n# Checkout previous known-good state\ngit checkout HEAD~1  # Go back 1 commit\n\n# Restart services with previous version\ndocker-compose up -d\n\n# Verify services are up\ndocker-compose ps\n```\n\n### Full Rollback (restore to Phase 4)\n\n```bash\n# Stop all services\ndocker-compose down\n\n# Checkout Phase 4 branch\ngit checkout feat/phase-4-cleanup\n\n# Rebuild and restart\ndocker-compose build\ndocker-compose up -d\n\n# Verify\ndocker-compose ps\n```\n\n## Common Issues & Solutions\n\n### Issue 1: \"Ollama connection refused\"\n\n**Symptoms**: Code-server logs show \"Cannot connect to ollama:11434\"\n\n**Solution**:\n```bash\n# Check ollama is running\ndocker-compose ps ollama\n\n# Check ollama logs\ndocker-compose logs ollama\n\n# Restart ollama\ndocker-compose restart ollama\nsleep 5\n\n# Verify connectivity from code-server\ndocker-compose exec code-server curl http://ollama:11434/api/tags\n```\n\n### Issue 2: \"No models available\"\n\n**Symptoms**: @ollama chat participant shows \"No models available\"\n\n**Solution**:\n```bash\n# Check models are loaded\ncurl http://localhost:11434/api/tags | jq '.models'\n\n# Pull default model\ncurl -X POST http://localhost:11434/api/pull -d '{\"name\":\"llama2:7b-chat\"}'\n\n# Wait for download to complete\nsleep 30\n\n# Verify model is loaded\ncurl http://localhost:11434/api/tags | jq '.models[] | select(.name==\"llama2:7b-chat\")'\n```\n\n### Issue 3: \"AI indexing not working\"\n\n**Symptoms**: Semantic search returns no results\n\n**Solution**:\n```bash\n# Check AI_INDEXING_ENABLED in .env\ngrep \"AI_INDEXING_ENABLED\" .env\n\n# Check backend logs for indexing errors\ndocker-compose logs code-server | grep -i \"index\"\n\n# Verify backend has access to ollama repo\nls -la ../ollama/backend/src/services/ai/\n\n# Restart code-server\ndocker-compose restart code-server\n```\n\n## Success Criteria\n\nPhase 5 deployment is successful when:\n\n✅ All Docker services start without errors  \n✅ Code-server is accessible and responsive  \n✅ Ollama is accessible
