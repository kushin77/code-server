# Copilot Chat VPN Testing Guide## Pre-Testing Checklist- ✅ Wireguard "web browser testing" profile configured (see screenshot)- ✅ Code-server deployed with navigator shim patch- ✅ Copilot extensions (github.copilot & github.copilot-chat) installed- ✅ product.json properly patched (trustedExtensionAuthAccess, no defaultChatAgent)- ✅ All containers healthy (code-server, oauth2-proxy, caddy)---## User Testing Procedure### Step 1: Connect to VPN1. Open Wireguard application2. Select the "web browser testing" profile (from your Wireguard config)3. Click "Activate" to connect4. Verify connected status shows ✓**Why this matters:** The VPN routes your traffic through the secure tunnel so you can access the private code-server deployment at `ide.kushnir.cloud`.### Step 2: Navigate to IDE1. Open your web browser2. Go to: `https://ide.kushnir.cloud`3. You should see a Google OAuth login (or skip if already cached)4. Complete Google OAuth if needed**Expected result:** Code-server web interface loads### Step 3: Hard Refresh (CRITICAL)1. Press `Ctrl+Shift+R` (Windows/Linux) or `Cmd+Shift+R` (Mac)2. This hard-refreshes the page AND forces the extension host to reinitialize3. Wait 10-15 seconds for the extension host to start**Why this matters:** A normal refresh (`F5`) only refreshes the UI. A hard refresh clears cache and reinitializes the extension host process, which is where our navigator shim patch takes effect.### Step 4: Verify No Error Messages1. Look at the VS Code interface2. **Watch for this error:** *"An error occurred while setting up chat. The extension 'GitHub.copilot-chat' cannot be installed because it was not found."*3. If you see it → ❌ The fix did not work (report this)4. If you don't see it → ✅ The fix is working**What to look for:** The error appears as a red box notification in the UI, typically in the top-center or bottom-right.### Step 5: Open Copilot Chat1. Press `Ctrl+Shift+I` (Windows/Linux) or `Cmd+Shift+I` (Mac)2. Alternatively: Click the Copilot icon in the left sidebar (chat bubble icon)3. The chat panel should open on the right side**Expected states:**- ✅ Chat panel opens with "Getting chat ready..." message- ✅ Chat panel opens with "Sign in with GitHub" button- ✅ Chat panel opens with message input field ready to use- ❌ Chat panel shows error about extension not found (failure)### Step 6: Authenticate with GitHub (if needed)1. If chat opens but shows "Sign in with GitHub" button:   - Click the button   - You'll be redirected to GitHub OAuth   - Authorize code-server to access your GitHub account   - You'll be redirected back to code-server2. If chat already shows message input field:   - You're already authenticated (from previous session)   - Skip this step**Note:** GitHub Copilot requires:- A GitHub account- An active Copilot subscription OR free trial access- Successful OAuth authorization### Step 7: Test Chat Functionality1. Once authenticated and chat panel is ready:2. Click in the message input field3. Type a test question, e.g.:   - "What is this repository about?"   - "How is the code-server deployed?"   - "Explain the Dockerfile patches"4. Press Enter to send5. Wait for Copilot to respond**Expected behavior:**- ✅ Chat accepts your message- ✅ Spinner shows "thinking..."- ✅ Response appears from Copilot- ✅ You can continue the conversation---## Troubleshooting### If you see the "cannot be installed" error:1. Check VPN is connected (status bar should show tunnel icon)2. Hard refresh the page again (`Ctrl+Shift+R`)3. Wait 20 seconds and try again4. If error persists:   - Open browser developer console (`F12`)   - Go to Console tab   - Look for error messages related to "navigator" or "PendingMigrationError"   - Screenshot and report the error### If chat opens but sign-in fails:1. Check GitHub account has Copilot access (Free or Pro)2. Try signing out and signing back in via GitHub browser3. Check that oauth2-proxy is passing GitHub tokens correctly### If chat opens but doesn't respond to messages:1. Wait 10 seconds (initial response can be slow)2. Check internet connection3. Verify GitHub token is still valid (may need to re-authenticate)---## Success Criteria✅ **All tests pass if:**1. No "cannot be installed" error appears2. Chat panel opens without errors3. GitHub OAuth completes successfully4. Chat accepts messages and responds---## Technical Details (for debugging)**What we fixed:**- VS Code 1.115.0 + Node.js 22.22.1 combination was throwing `PendingMigrationError` when extensions accessed the `navigator` object- Copilot Chat 0.43.x uses Zod schema validation which accesses `navigator` at module load time- This caused an immediate crash at extension activation, triggering a broken reinstall cycle- **Solution:** Remove the throwing `Object.defineProperty(globalThis,"navigator"...)` shim at build time via Perl patch in Dockerfile**Files involved:**- `Dockerfile.code-server` - Contains the navigator shim patch ([line 20-39](../../Dockerfile.code-server#L20-L39))- `docker-compose.yml` - Sets `NODE_OPTIONS=--enable-source-maps` for debugging- `scripts/test-deployment.sh` - Validates all patches are in place- `/usr/lib/code-server/lib/vscode/product.json` - Patched to add `github.copilot-chat` to `trustedExtensionAuthAccess`**How to verify the fix is installed:**Inside the code-server container:```bash# Check navigator shim is removed (count should be 0)grep -c "navigator is now a global" \  /usr/lib/code-server/lib/vscode/out/vs/workbench/api/node/extensionHostProcess.js# Output: 0# Check extensions are installed/usr/bin/code-server --list-extensions --extensions-dir \  /home/coder/.local/share/code-server/extensions | grep copilot# Output:# github.copilot# github.copilot-chat```---## Testing CompleteOnce you've completed Steps 1-7 above, the deployment is verified.Report results to confirm fix is working:- ✅ No error, chat works → **Fix successful**- ❌ Error appears → Debug needed (see Troubleshooting)
