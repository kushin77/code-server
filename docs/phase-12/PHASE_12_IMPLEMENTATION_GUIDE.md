Event published to Kafka (2ms)```**Replication Path** (50-100ms):```Master Region (PostgreSQL BDR)  → Event log capture (5ms)  → Kafka pub-sub (20ms)  → Secondary regions (25-50ms)  → CRDT merge (5ms)  → PostgreSQL apply (5ms)```## Consistency Model**Layer**: Eventual consistency with CRDT guarantees**Convergence**: <200ms (99th percentile)**Data Loss**: Zero (WAL + synchronous commit)**Availability**: 99.99% (fails over automatically)## Failure Scenarios### Single Region Failure**Time**: <30s detection + recovery**Impact**: Users routing to next nearest region**Data**: No loss (multi-primary replication)### Network Partition**Detection**: <30s (health checks)**Automatic**: Circuit breaker activates**Recovery**: CRDT merge on reconnect### Cascading Failures (2+ regions)**Strategy**: Quorum-based routing**Minimum**: 1 region must be operational**Recovery**: Cross-region event replay## Key Decisions| Decision | Rationale | Trade-off ||----------|-----------|-----------|| Multi-primary (not primary-secondary) | Higher availability | Requires conflict resolution || Eventual consistency (not linearizable) | Distributed systems reality | Temporary inconsistencies || CRDT-based merging (not OT) | Commutative & idempotent | Limited to certain data types || Event streaming layer (in addition to replication) | Ordering guarantees | Additional complexity |## Performance Targets| Metric | Target | Measurement ||--------|--------|-------------|| Local latency | <150ms p99 | Single region || Global latency | <250ms p99
