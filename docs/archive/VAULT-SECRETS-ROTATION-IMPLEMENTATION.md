# VAULT SECRETS ROTATION IMPLEMENTATION (#356)

## Executive Summary

Implemented automatic PostgreSQL credential rotation via HashiCorp Vault with:
- **Dynamic secret generation** via Vault database roles
- **Automatic 45-minute rotation** with systemd timer + cron fallback
- **TTL-based lifecycle**: 1 hour default, 24 hours maximum
- **Zero-downtime credential refresh** using credential path injection
- **Prometheus monitoring** for rotation failures and credential expiration

## Implementation Details

### 1. Vault Database Backend Configuration

**Location**: `terraform/phase-8-vault-secrets-rotation.tf`

Configures Vault to manage PostgreSQL authentication:

```bash
vault write database/config/postgresql \
  plugin_name=postgresql-database-plugin \
  connection_url="postgresql://{{username}}:{{password}}@postgres:5432/postgres" \
  username=postgres \
  password=$POSTGRES_PASSWORD
```

### 2. Dynamic Roles

#### Role: `code-server` (Application)
- **TTL**: 1 hour default, 24 hours maximum
- **Permissions**: SELECT, INSERT, UPDATE, DELETE on all tables
- **Username pattern**: `cs-{timestamp}` to prevent collisions
- **Use case**: Application credentials with full CRUD access

#### Role: `readonly` (Analytics/Monitoring)
- **TTL**: 2 hours default, 48 hours maximum
- **Permissions**: SELECT only on all tables
- **Use case**: Read-only access for monitoring and analytics

### 3. Credential Rotation Script

**Location**: `scripts/setup-vault-secrets-rotation.sh`

**Execution Flow**:
```
1. Fetch new credentials from Vault
   └─ Credentials auto-generated by Vault role
2. Save to /var/run/secrets/vault-postgres/.postgres-creds.env
   └─ Permissions: 0600 (owner-only readable)
3. Notify containers (docker exec or file watcher)
   └─ Application reloads credentials
4. Log rotation event + metrics
   └─ Prometheus scrapes /metrics endpoint
```

**Trigger Mechanisms**:
- **Primary**: Systemd timer (vault-rotate-credentials.timer)
  - Runs every 45 minutes
  - Persistent across reboots
  - Automatic retry on failure
- **Fallback**: Cron job (/etc/cron.d/vault-postgres-rotation)
  - Backup rotation mechanism
  - Logs to /var/log/vault-rotation.log

### 4. Credential Injection

Application reads credentials from `/var/run/secrets/vault-postgres/.postgres-creds.env`:

```python
# application/db.py
import os
from pathlib import Path

CREDS_PATH = Path("/var/run/secrets/vault-postgres/.postgres-creds.env")

def load_credentials():
    if CREDS_PATH.exists():
        # Load from Vault-rotated credentials
        with open(CREDS_PATH) as f:
            for line in f:
                if line.startswith("POSTGRES_"):
                    key, val = line.strip().split("=", 1)
                    os.environ[key] = val
    # Fall back to env vars if file not present
    return {
        "user": os.environ["POSTGRES_USER"],
        "password": os.environ["POSTGRES_PASSWORD"],
        "host": os.environ["POSTGRES_HOST"],
        "port": os.environ["POSTGRES_PORT"],
    }

DATABASE_URL = load_credentials()
```

### 5. Monitoring & Alerts

**Prometheus Rules**: `/etc/prometheus/rules/vault-rotation-alerts.yml`

Alerts fired:
- `VaultCredentialRotationFailed` - Rotation script exited with error
- `VaultCredentialRotationLate` - No rotation in >1 hour (warning)
- `VaultCredentialExpiringWithin1Hour` - Credentials approaching TTL

**Metrics Collected**:
```prometheus
vault_rotation_failures_total     # Count of failed rotations
vault_last_rotation_timestamp     # Unix timestamp of last rotation
vault_credential_expiration_unix  # When current credentials expire
```

## Security Considerations

### Secret Storage

- **Credentials**: Stored in `/var/run/secrets/vault-postgres/.postgres-creds.env`
  - File permissions: 0600 (owner-only)
  - Mounted on tmpfs (memory-only, not persisted to disk)
  - Automatically cleared on reboot

- **Vault Token**: Passed via `VAULT_TOKEN` environment variable
  - Should use temporary token with limited lifetime
  - Recommend: 1-hour token generated by Terraform
  - Never commit to Git or Docker images

### Authentication

- **Vault**: mTLS + tokens (configure `VAULT_TOKEN` rotation)
- **PostgreSQL**: Dynamic user creation by Vault
  - Each user has unique username (includes timestamp)
  - Passwords auto-generated (cryptographically strong)
  - Users dropped automatically on TTL expiry

### Least Privilege

- **Application user** (`code-server` role):
  - Only SELECT, INSERT, UPDATE, DELETE
  - No CREATE/DROP/ALTER permissions
  - Scoped to public schema

- **Read-only user** (`readonly` role):
  - SELECT only
  - No write permissions
  - For monitoring/analytics

## Troubleshooting

### Issue: "Failed to fetch credentials from Vault"

**Causes**:
1. `VAULT_TOKEN` expired or invalid
2. Vault not accessible at `VAULT_ADDR`
3. Vault database backend not configured

**Fix**:
```bash
# Verify Vault connectivity
vault status

# Check database backend exists
vault list database/roles/

# Re-generate temporary token
vault token create -ttl=1h
export VAULT_TOKEN=<new_token>

# Run rotation script with debug output
bash -x scripts/setup-vault-secrets-rotation.sh
```

### Issue: "Credentials not rotating"

**Causes**:
1. Systemd timer not active
2. Cron job not installed
3. Rotation script has permission error

**Fix**:
```bash
# Check systemd timer
systemctl status vault-rotate-credentials.timer
systemctl status vault-rotate-credentials.service

# Enable if disabled
systemctl enable vault-rotate-credentials.timer
systemctl start vault-rotate-credentials.timer

# Check cron job
cat /etc/cron.d/vault-postgres-rotation
ps aux | grep rotate-vault

# View logs
journalctl -u vault-rotate-credentials.service -f
tail -f /var/log/vault-rotation.log
```

### Issue: "Application cannot connect to PostgreSQL"

**Cause**: Credentials file not being updated or application not reloading

**Fix**:
```bash
# Verify credentials file exists and is readable
ls -la /var/run/secrets/vault-postgres/
cat /var/run/secrets/vault-postgres/.postgres-creds.env

# Force credential rotation
/usr/local/bin/rotate-vault-credentials.sh

# Verify PostgreSQL user exists
psql -U postgres -c "\du" | grep cs-

# Restart application container
docker restart code-server
```
