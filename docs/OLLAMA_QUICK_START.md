# Ollama Quick Deployment Guide**Time to complete**: 5-10 minutes (+ 30-60 min for model pulling)## Step 1: Deploy Infrastructure (5 min)```bashcd /path/to/code-server-enterprise# Start all servicesmake deploy# Verify everything is runningmake status```Expected output:```üê≥ Container Status:code-server        Up 1m (healthy)oauth2-proxy       Up 1m (healthy)caddy              Up 1m (healthy)ollama             Up 1mollama-init        Up 1m```## Step 2: Pull Models (30-60 min)This downloads the elite LLM models. Run in the background:```bash# Pull all 4 modelsmake ollama-pull-models# Or monitor in real timemake ollama-logs```Models being pulled:- `llama2:70b-chat` (31GB) - Elite reasoning- `codegemma` (9GB) - Code analysis- `neural-chat` (13GB) - General chat- `mistral` (4GB) - Fast inference**Tip**: You can start using code-server while models are downloading.## Step 3: Initialize Repository Indexing (1-2 min)```bashmake ollama-init```This creates semantic embeddings of your codebase for context retrieval.## Step 4: Start Using Ollama (30 sec)1. Open **code-server** in browser2. Open the **Chat** view (Ctrl+Alt+I)3. Type `@ollama` followed by your prompt:```@ollama explain this function@ollama generate tests for this file@ollama refactor this code```## Verify Installation```bash# Check Ollama healthmake ollama-health# List available modelsmake ollama-list# Show Ollama statusmake ollama-status```## Troubleshooting During Model Pulling### "out of disk space"- Check available disk: `docker system df`- Clear space or reduce model count- Pull just the fast model: `docker compose exec ollama ollama pull mistral`### "Ollama not responding"- Wait a bit longer (models are still downloading)- Check logs: `make ollama-logs`- Restart: `docker compose restart ollama`### "CPU maxed out"- Normal during model pulling- Models will cache once fully downloaded- Subsequent operations are faster## Post-Deployment Optimization### For Development (Minimal Resources)```bash# Stop unused modelsdocker compose exec ollama ollama rm neural-chatdocker compose exec ollama ollama rm codegemma# Keep only: mistral (fast) + llama2 (best quality)```### For Production (Maximum Quality)```bash# Use the 70B model as default# Edit .vscode/settings.json:# "ollama.defaultModel": "llama2:70b-chat"# Increase context window for longer code# "ollama.contextWindow": 8192```### Enable GPU Acceleration (If Available)```bash# Edit docker-compose.yml:# ollama:#   environment:#     - OLLAMA_NUM_GPU=1# Rebuild and restartdocker compose builddocker compose up -d```## Usage Examples### Code Analysis```@ollama What are the performance bottlenecks in this code?```### Test Generation```@ollama Generate comprehensive
