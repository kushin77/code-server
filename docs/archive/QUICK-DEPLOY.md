Update terraform.tfvars# 2. Re-apply: terraform apply# 3. Update .env: docker-compose up -d# 4. Restart: docker-compose restart oauth2-proxy```### Copilot Chat not working```bash# Check code-server logsdocker exec code-server grep -i copilot /home/coder/.local/share/code-server/logs/*# Common issues:# - GitHub PAT invalid/expired# - Wrong scopes on PAT (needs: read:user, user:email)# - Network blocked to GitHub API# Fix:# 1. Verify GITHUB_TOKEN in .env# 2. Check: curl -I https://api.github.com (network check)# 3. Restart code-server: docker-compose restart code-server```### Ollama not loading models```bash# Check ollama logsdocker logs ollama# Common issues:# - Model not downloaded: First startup downloads 13GB+ model# - Disk space: Verify available disk (needs 50GB+ for large models)# - GPU not available: Ollama falls back to CPU (slower)# Check model status:curl http://localhost:11434/api/tags | jq# Pull model manually:docker exec ollama ollama pull llama2:70b-chat# Monitor progress:docker logs -f ollama```### High memory usage```bash# Check resource usagedocker stats# If Ollama using too much:# - Switch to smaller model: ollama pull mistral (7B instead of 70B)# - Update terraform.tfvars: ollama_default_model = "mistral:7b"# - Re-apply: terraform apply && docker-compose restart ollama# Monitor during load:watch -n 1 'docker stats --no-stream'```---## 📊 Production Monitoring### Essential Metrics to Monitor```bash# Container health (every 5 minutes)while true; do  clear  date  docker-compose ps  echo "---"  docker stats --no-stream  sleep 300done# Log monitoring (tail important containers)docker-compose logs -f code-server oauth2-proxy caddy ollama# Verify services still respondingwatch -n 30 'curl -s http://localhost:11434/api/tags > /dev/null && echo "✅ Ollama OK" || echo "❌ Ollama FAILED"'```### Alerts to Set Up- [ ] Code-server not responding (no response for 5 minutes)- [ ] Memory usage > 80%- [ ] Disk usage > 90%- [ ] OAuth login failures (spike in 401s)- [ ] Ollama inference errors- [ ] Container auto-restart (check if containers restarting frequently)---## 🔄 Rollback ProcedureIf something goes wrong:```bash# Stop all servicesdocker-compose down# Remove volumes (CAUTION: deletes persistent data)docker-compose down -v# Reset infrastructureterraform destroy -auto-approve# Restart from Phase 1# (Follow full deployment again)```**Less destructive rollback:**```bash# Just stop services (keep data)docker-compose stop# Fix issue in terraform.tfvars or docker-compose.yml# Restartterraform applydocker-compose up -d```---## 📚 Full Documentation References- **[DEPLOYMENT_CHECKLIST.md](DEPLOYMENT_CHECKLIST.md)** -
