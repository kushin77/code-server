partition   - Ensures users in any region can always write   - Eventual consistency within bounded time2. **CRDT-Based Merging**   - All application data represented as CRDTs   - Merge function is commutative, associative, idempotent   - No coordination needed for conflict resolution3. **Event-Driven Sync**   - Changes published as immutable events   - Events contain full operation context   - Replayed idempotently at all sites4. **Local-First Performance**   - Read/write latency = local cluster latency (< 50ms)   - No remote calls in critical path   - Async replication hidden from users###
