## Automated Deployment (Single Command)

```bash
cd /path/to/code-server-enterprise/scripts
./automated-deployment-orchestration.sh
```

**That's it.** The orchestration script handles:

1. ✅ **Environment Validation** - Checks SSH, Docker, dependencies
2. ✅ **Configuration Generation** - Generates .env with secure credentials
3. ✅ **Certificate Management** - Provisions self-signed bootstrap certs, configures Let's Encrypt
4. ✅ **DNS Configuration** - Updates CloudFlare DNS records (if credentials provided)
5. ✅ **Service Deployment** - Pulls images, starts containers, waits for health checks
6. ✅ **Deployment Validation** - Verifies all services are running
7. ✅ **Summary Reporting** - Generates deployment report

## Step-by-Step Automation Details

### Step 1: Validate Environment

**Automated by:** `validate_environment()` in orchestration script

```bash
# Checks performed automatically:
- Local command availability (ssh, scp, docker, docker-compose, openssl, curl, jq)
- SSH connectivity to deploy host
- Docker availability on target host
- Access permissions
```

**No action required.** Script validates and exits on any prerequisite failure.

---

### Step 2: Generate Production Configuration

**Automated by:** `automated-env-generator.sh`

```bash
# Automatically generates:
- .env file with secure environment variables
- Random 32-byte credentials for Code-Server
- Random 32-byte secrets for OAuth2-Proxy
- Random 16-byte passwords for Redis
- Domain and host configuration
- OAuth client configuration (if env vars provided)
```

**Generated File:** `code-server-enterprise/.env.production`

```env
# Example generated configuration (actual values differ)
DOMAIN=ide.kushnir.cloud
DEPLOY_HOST=192.168.168.31
DEPLOY_ENV=production
CODE_SERVER_PASSWORD=<auto-generated-base64>
OAUTH2_PROXY_COOKIE_SECRET=<auto-generated-base64>
REDIS_PASSWORD=<auto-generated-base64>
```

---

### Step 3: Configure DNS (Optional, Fully Automated)

**Automated by:** `automated-dns-configuration.sh`

**Requirements:**
- `CLOUDFLARE_API_TOKEN` environment variable set
- `CLOUDFLARE_ZONE_ID` environment variable set

**Automatically performs:**
- Validates CloudFlare credentials
- Creates/updates A record for domain
- Creates/updates wildcard DNS records
- Verifies DNS propagation
- Saves configuration for audit

**If CloudFlare credentials not provided:** DNS configuration skipped gracefully (non-blocking)

```bash
# To enable DNS automation:
export CLOUDFLARE_API_TOKEN="<api-token-from-dash.cloudflare.com>"
export CLOUDFLARE_ZONE_ID="<zone-id-from-domain-dashboard>"
```

---

### Step 4: Prepare Deployment Files

**Automated by:** Orchestration script

**Copies to target host:**
- docker-compose.yml (IaC orchestration)
- Caddyfile (reverse proxy config)
- .env file (runtime configuration)

**Target location:** `/home/akushnir/code-server-immutable-<timestamp>/`

---

### Step 5: Deploy Services

**Automated by:** Docker Compose on target host

**Executed automatically:**
```bash
cd ${DEPLOYMENT_DIR}
docker-compose pull          # Update images
docker-compose up -d         # Start all 5 services
docker-compose ps            # Verify startup
```

**Services deployed:**
1. **Caddy** - Reverse proxy, automatic HTTPS, health monitoring
2. **Code-Server** - IDE backend, code execution, extensions
3. **Ollama** - LLM backend, model inference
4. **OAuth2-Proxy** - Authentication layer, session management
5. **Redis** - Cache and session store, persistence

**Health checks:** All services include automatic health checks. Docker will restart unhealthy services.

---

### Step 6: Validate Deployment

**Automated by:** Health check validation scripts

**Automatically verifies:**
```bash
docker-compose config --quiet  # Configuration validity
docker-compose ps              # Service status
curl health endpoints          # Service availability
```

All validation is automated. Script reports success/failure for each service.

## Security Considerations

### Credentials Management

- **Generated via:** OpenSSL cryptographically secure random generation
- **Storage:** Mounted as Docker secrets, stored on host as .env (chmod 600)
- **Never logged:** Credentials excluded from output/logs
- **Rotation capability:** Update .env and run `docker-compose restart`

### Certificate Management

- **Bootstrap:** Self-signed certificates for initial deployment
- **Production:** Automatic Let's Encrypt provisioning via Caddy
- **TLS 1.3:** Enforced via Caddyfile configuration
- **Renewal:** Automatic via Caddy ACME client
- **Storage:** Persisted in Docker volumes with backups

### Network Security

- **Isolation:** Services on isolated Docker network (10.0.8.0/24)
- **Firewall:** Only ports 80/443 exposed (via Caddy)
- **Internal:** Service-to-service communication via internal network
- **Proxy:** No direct service access; all through Caddy

## Disaster Recovery

### Complete Rebuild

If entire deployment fails, force rebuild:

```bash
# Set environment variables
export DOMAIN="ide.kushnir.cloud"
export DEPLOY_HOST="192.168.168.31"
export DEPLOY_USER="akushnir"

# Cleanup (optional, removes old deployment)
ssh akushnir@192.168.168.31 "docker-compose -f /home/akushnir/code-server-immutable-*/docker-compose.yml down -v"

# Redeploy with fresh IaC
./automated-deployment-orchestration.sh
```

This produces a clean, identical deployment from scratch.

### Restore from Backup

```bash
ssh akushnir@192.168.168.31 << 'EOF'
cd /home/akushnir/code-server-immutable-*/

# Stop running services
docker-compose down

# Restore .env and docker-compose.yml from backup
cp /home/akushnir/.backups/latest/.env .env
cp /home/akushnir/.backups/latest/docker-compose.yml docker-compose.yml

# Restart with backed-up config
docker-compose up -d

# Verify
docker-compose ps
EOF
```
