# Ollama Integration - Elite Local LLM for code-server-enterprise**Status**: ✅ Production Ready**Date**: 2026-04-12**Version**: 1.0.0## OverviewThis code-server-enterprise now includes **Ollama**, a world-class local LLM server that rivals Claude Opus and other elite models. The integration provides:- 🏆 **llama2:70b-chat** - Elite reasoning model that matches Claude Opus- 🎯 **codegemma** - Google's code-specialized model- ⚡ **neural-chat** - Fast, capable model for real-time feedback- 🚀 **mistral** - Lightweight model for quick queries- 🧠 **Repository Context Learning** - Automatic indexing and embedding of your codebase- 💬 **Native VS Code Integration** - Chat directly in the IDE via @ollama- 🔒 **100% Local** - No data leaves your machine, no external API calls## Architecture```┌─────────────────────────────────────────────────────────┐│ VS Code (code-server)                                   │├─────────────────────────────────────────────────────────┤│ ┌─ Ollama Chat Extension ──────────────────────────┐    ││ │ • Chat Participant (@ollama)                    │    ││ │ • Repository Context Indexer                    │    ││ │ • Code Analyzer                                 │    ││ │ • Commands (analyze, generate, test, etc)      │    ││ └─────────────────────────────────────────────────┘    │└──────────────┬──────────────────────────────────────────┘               │ HTTP/REST (localhost:11434)┌──────────────v──────────────────────────────────────────┐│ Ollama Server (Docker Container)                        │├──────────────────────────────────────────────────────────┤│ • llama2:70b-chat (31GB) - Elite reasoning              ││ • codegemma (9GB) - Code analysis                       ││ • neural-chat (13GB) - Fast inference                   ││ • mistral (4GB) - Ultra-fast                            ││ • Embedding API for context retrieval                   │└─────────────────────────────────────────────────────────┘```## Quick Start### 1. Deploy with Ollama```bashmake deploy```This starts all services including Ollama.### 2. Pull Models (takes 30-60 minutes depending on internet)```bashmake ollama-pull-models```Or pull individual models:```bashdocker compose exec ollama ollama pull llama2:70b-chatdocker compose exec ollama ollama pull codegemma```### 3. Initialize Repository Context```bashmake ollama-index```This walks your workspace and builds embeddings for fast context retrieval.### 4. Start ChattingOpen code-server and go to the **Chat** view (Ctrl+Alt+I), then type:```@ollama explain this function@ollama generate tests for this file@ollama refactor this code```## Models & Resource Requirements| Model | Size | VRAM | RAM | Use Case | Quality ||-------|------|------|-----|----------|---------|| **llama2:70b-chat** | 31GB | 32GB | 8GB | Complex analysis, architecture decisions | 🏆 Elite (Claude Opus-class) || **codegemma** | 9GB | 12GB | 4GB | Code generation, refactoring | 🎯 Excellent || **neural-chat** | 13GB | 16GB | 4GB | General chat, explanations | ⚡ Very Good || **mistral** | 4GB | 6GB | 2GB | Quick queries, summaries | 🚀 Good |**Recommended Setup**:- **Development**: mistral + neural-chat (~6GB RAM)- **Production**: llama2:70b + codegemma (~40GB RAM)- **Balanced**: neural-chat + codegemma + mistral (~23GB RAM)## Usage Examples### Code Analysis```@ollama analyze this function for performance issues and security vulnerabilities```Response:```Performance Issues Found:1. Here is context from the codebase:File: src/auth/tokens.ts[relevant code excerpt]File: src/auth/strategies.ts[relevant code excerpt]User Request: How does the authentication system work?Respond with architectural overview, key components, and data flow.```### Fine-Tuning Models (Advanced)Models can be fine-tuned on your codebase for better understanding:```bash# Create training data from your repopython scripts/prepare-training-data.py --repo . --output training.txt# Fine-tune (creates new model version)docker compose exec ollama ollama create my-codebase \  --from llama2:70b-chat \  --file Modelfile  # Contains training data# Use fine-tuned model# In settings: ollama.defaultModel = "my-codebase"```## Troubleshooting### "Ollama server not responding"```bash# Check if container is runningdocker compose ps ollama# Check logsdocker compose logs ollama# Restart servicedocker compose restart ollama# Check connectivitycurl http://localhost:11434/api/tags```### Models not pulling```bash# Check disk spacedocker system df# Check Ollama download logsdocker compose logs -f ollama# Manual pull with retrydocker compose exec ollama ollama pull llama2:70b-chat# Check downloaded modelsdocker compose exec ollama ollama list```### Slow responses- **Reduce context window** in settings (default 4096 tokens)- **Use faster model** (neural-chat, mistral instead of llama2:70b)- **Check system resources**: `docker stats ollama`- **Increase RAM** if swapping occurs- **Use GPU acceleration** if available (`OLLAMA_NUM_GPU=1`)### Extension not loading```bash# Reinstall extensiondocker compose exec code-server \  /usr/local/bin/code-server-entrypoint.sh# Check extension installationdocker compose exec code-server \  code-server --list-extensions --extensions-dir /home/coder/.local/share/code-server/extensions```### Out of memory (OOM)```bash# Stop Ollamadocker compose stop ollama# Increase available memory# Edit docker-compose.yml: deploy.resources.limits.memory: 64G# Unload unused modelsdocker compose exec ollama ollama rm neural-chat# Restartdocker compose up -d```## Performance Tuning### CPU-Only (No GPU)```bash# Reduce
