# Phase 3: Integration Update Guide\n\n## Overview\n\nThis guide explains how to integrate kushin77/ollama AI services into code-server-enterprise.\n\nAfter PR #103 merges in kushin77/ollama, code-server-enterprise can reference the ollama services via:\n1. **NPM Workspace dependencies** for development\n2. **Docker Compose services** for production\n3. **Git submodule or npm package** for stable releases\n\n## Integration Approach\n\n### Option 1: NPM Workspaces (Recommended for Development)\n\nUpdate `package.json` in code-server-enterprise:\n\n```json\n{\n  \"name\": \"code-server-enterprise\",\n  \"workspaces\": [\n    \".\",\n    \"../ollama\"\n  ]\n}\n```\n\nBenefit: Shared node_modules, single npm install, symbolic links to ollama code.\n\n```bash\n# After cloning both repos in same parent directory:\ncd code-server-enterprise\nnpm install  # Installs dependencies for both repos\n```\n\n### Option 2: Docker Compose Integration (Production)\n\nUpdate `docker-compose.yml` to include ollama services:\n\n```yaml\nversion: '3.9'\n\nservices:\n  code-server:\n    image: code-server:latest\n    environment:\n      OLLAMA_ENDPOINT: http://ollama:11434\n      AI_INDEXING_ENABLED: \"true\"\n      AI_INDEX_CHUNK_SIZE_TOKENS: \"800\"\n    depends_on:\n      - ollama\n    # Uncomment to load ollama extension:\n    # volumes:\n    #   - ./extensions/ollama-chat:/home/coder/.local/share/code-server/extensions/ollama-chat:ro\n\n  ollama:\n    image: ollama:latest\n    ports:\n      - \"11434:11434\"\n    volumes:\n      - ollama-data:/root/.ollama\n    environment:\n      OLLAMA_MODELS: \"llama2:70b-chat codegemma neural-chat mistral\"\n    command: serve\n\nvolumes:\n  ollama-data:\n```\n\n### Option 3: Git Submodule (Hybrid)\n\nFor stable, pinned versions:\n\n```bash\ncd code-server-enterprise\ngit submodule add https://github.com/kushin77/ollama.git ollama-repo\ngit submodule update --remote\n```\n\nThen reference:\n```bash\n# Development\ncd ollama-repo/extensions/ollama-chat\nnpm link\n\ncd ../../..\nnpm link ollama-chat-vscode\n```\n\n## File Structure After Integration\n\n```\ncode-server-enterprise/\n‚îú‚îÄ‚îÄ extensions/\n‚îÇ   ‚îú‚îÄ‚îÄ ollama-chat/  ‚Üê symlink to ../ollama/extensions/ollama-chat (if using workspaces)\n‚îÇ   ‚îî‚îÄ‚îÄ ...\n‚îú‚îÄ‚îÄ (code-server ‚Üí ollama)\n\n| Variable | Default | Purpose | Set In |\n|----------|---------|---------|--------|\n| `OLLAMA_ENDPOINT` | `http://ollama:11434` | Ollama server address | docker-compose.yml or .env |\n| `AI_INDEXING_ENABLED` | `true` | Enable semantic indexing | docker-compose.yml or .env |\n| `AI_EGRESS_ENABLED` | `false` | Allow HuggingFace egress | .env (secure) |\n| `HF_API_TOKEN` | `` | HuggingFace API key | .env (secure) |\n| `AI_INDEX_CHUNK_SIZE_TOKENS` | `800` | Semantic chunk size | docker-compose.yml |\n| `AI_INDEX_OVERLAP_TOKENS` | `120` | Overlap between chunks | docker-compose.yml |\n| `AI_INDEX_QUEUE_SIZE` | `1000` | Async queue depth | docker-compose.yml |\n\n### Backend Service Integration\n\ncode-server's backend can now import AI services:\n\n```typescript\n// In code-server backend (e.g., backend/src/routes/ai.ts)\nimport { \n  RepositoryIndexer, \n  AIRouter, \n  type RepositoryFile \n} from '../services/ai';\n\nconst indexer = new RepositoryIndexer({\n  chunkSizeTokens: 800,\n  chunkOverlapTokens: 120,\n  maxQueueSize: 1000,\n});\n\n// Index workspace\nawait indexer.indexRepository(workspaceFiles);\n\n// Route AI requests\nconst router = new AIRouter();\nconst route = router.route({\n  task: 'chat',\n  prompt: userPrompt,\n});\n```\n\n### Extension Integration\n\nOllama extension in VS Code will:\n1. installed:\n\n```bash\ncd code-server-enterprise\nrm -rf node_modules package-lock.json\nnpm install\nls ../ollama  # Should exist\n```\n\n### \"Ollama server not responding\"\n\n**Issue**: Backend can't reach ollama service  \n**Solution**: Check docker-compose and networking:\n\n```bash\ndocker-compose ps\ndocker-compose logs ollama\ngrep OLLAMA_ENDPOINT .env\ncurl $OLLAMA_ENDPOINT/api/tags\n```\n\n### \"Extension not loaded in VS Code\"\n\n**Issue**: @ollama chat participant not appearing  \n**Solution**: Check extension installation:\n\n```bash\nls -la extensions/ollama-chat/\nls ../ollama/extensions/ollama-chat/\nnpm run build:extension\n```\n\n## Rollback\n\nIf integration issues occur:\n\n```bash\n# Revert to version before ollama
