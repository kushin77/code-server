**Resolution**:

- Consolidated terraform.tfvars (merged phase-14 config)

**Impact**: Codebase now compliant with copilot-instructions.md "Prime Directive: No Timelines"

### 2. CADDY HEALTH CHECK FAILURES

**Problem**: Caddy container unhealthy for 20+ minutes
```
Error: ambiguous site definition: code-server.192.168.168.31.nip.io
Error: auto_https value invalid (set to 'off' without proper syntax)
```

**Root Cause Analysis**:
- Caddyfile templates used variable substitution: `code-server.{$DOMAIN:kushnir.cloud}` 
- When DOMAIN unset, created invalid site definition
- auto_https syntax error: `auto_https off` (invalid)

**Resolution Applied**:
- Rewrote Caddyfile with explicit on-prem configuration
- Removed ambiguous site definitions (no variable interpolation)
- Fixed global auto_https: `auto_https off` (proper syntax)
- Added explicit health check endpoint at `:80` with `/healthz` path
- Consolidated 11 Caddyfile variants → 1 semantic configuration

**Result**:
```
BEFORE: caddy          Up 20 minutes (unhealthy)
AFTER:  caddy          Up 2 seconds (health: starting) → Up 20 seconds (healthy)
```
